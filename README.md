## 📖 Sobre o projeto

O **Sistema Acadêmico GAIO**, acrônimo de **Gerenciador Acadêmico Institucional Online**, é um sistema de controle acadêmico desenvolvido para instituições públicas de ensino superior. O projeto visa modernizar, centralizar e facilitar a gestão de dados acadêmicos, oferecendo uma interface intuitiva para alunos, professores e o setor administrativo, com base nos princípios da **Interação Humano-Computador (IHC)** e **Usabilidade**.

Este sistema foi desenvolvido como Trabalho de Conclusão de Curso (TCC) no curso de Tecnologia em Análise e Desenvolvimento de Sistemas na **FAETERJ Rio de Janeiro**.

<br>

## ✨ Funcionalidades do sistema

A arquitetura do GAIO é organizada em **módulos** que agrupam processos e atores em comum, como:

* **Gestão de Usuários:** Controle centralizado de acessos, permissões e cadastro de discentes e docentes.
* **Gestão de Cursos:** Administração das diretrizes de ensino, englobando as matrizes curriculares e os períodos letivos.
* **Gestão de Espaços e Relatórios:** Gerenciamento da infraestrutura física da unidade e emissão de dados.
* **Gestão de Turmas:** Ponto de convergência que unifica cursos, usuários e espaços em uma estrutura letiva.
* **Gestão de Inscrições, Frequências e Avaliações:** Controle das rotinas diárias vinculadas às turmas, registrando a alocação, frequência e o desempenho acadêmico dos alunos.

## 🤝 Como contribuir

O GAIO é um projeto nascido no ambiente acadêmico e contribuições da comunidade, sejam elas para correção de bugs, melhorias de segurança ou novas funcionalidades, são muito bem-vindas!

Para contribuir com o código-fonte da aplicação, siga o fluxo abaixo:

### 1. Preparando o ambiente

* Faça um *fork* deste repositório para a sua conta no GitHub.
* Clone o seu *fork* para a sua máquina local:

```bash
git clone https://github.com/ThiMoreira00/GAIO.git
```

### 2. Desenvolvendo a sua contribuição

* Crie uma *branch* específica para a sua alteração, utilizando um nome descritivo:

```bash
git checkout -b feature/minha-nova-funcionalidade
```

* Ou para correções de bugs:
```bash
git checkout -b bugfix/correcao-calculo-notas
```

* Realize as alterações no código.
* Faça o *commit* das suas alterações. Recomendamos o uso do padrão [Conventional Commits](https://www.conventionalcommits.org/pt-br/v1.0.0/) para manter o histórico limpo (ex: feat:, fix:, docs:, refactor:):

```bash
git commit -m "feat: adiciona botão de exportação de histórico para PDF"
```

### 3. Enviando as alterações

* Envie a sua *branch* para o seu repositório remoto (*fork*):

```bash
git push origin feature/minha-nova-funcionalidade
```

* Acesse a página do repositório original do GAIO no GitHub e abra um *Pull Request* (PR).
* Descreva detalhadamente o que foi alterado, qual problema a sua contribuição resolve e anexe capturas de tela, se houver mudanças visuais.
